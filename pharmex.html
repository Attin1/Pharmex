⚙️ Supabase Kurulum Rehberi

Pharmex'i kullanmak için Supabase projenizi aşağıdaki adımlara göre ayarlayın.

1. Sayfanın en üstündeki CONFIG bölümünü doldurun:

const SUPABASE_URL = 'https://XYZ.supabase.co' const SUPABASE_ANON_KEY = 'eyJh...'

2. Supabase SQL Editor'de çalıştırın:

-- Tablolar CREATE TABLE profiles (id UUID PRIMARY KEY REFERENCES auth.users(id), email TEXT, role TEXT DEFAULT 'user', full_name TEXT, created_at TIMESTAMPTZ DEFAULT NOW()); CREATE TABLE pharmacies (id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES profiles(id), name TEXT NOT NULL, address TEXT, latitude FLOAT, longitude FLOAT, phone TEXT, is_active BOOL DEFAULT TRUE, created_at TIMESTAMPTZ DEFAULT NOW()); CREATE TABLE medicines (id UUID DEFAULT gen_random_uuid() PRIMARY KEY, name TEXT NOT NULL, barcode TEXT UNIQUE, manufacturer TEXT, created_at TIMESTAMPTZ DEFAULT NOW()); CREATE TABLE stock (id UUID DEFAULT gen_random_uuid() PRIMARY KEY, pharmacy_id UUID REFERENCES pharmacies(id) ON DELETE CASCADE, medicine_id UUID REFERENCES medicines(id) ON DELETE CASCADE, quantity INT DEFAULT 0, updated_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(pharmacy_id, medicine_id)); CREATE TABLE activity_logs (id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES profiles(id), action TEXT, details JSONB DEFAULT '{}', created_at TIMESTAMPTZ DEFAULT NOW()); CREATE TABLE search_analytics (id UUID DEFAULT gen_random_uuid() PRIMARY KEY, medicine_name TEXT, searched_at TIMESTAMPTZ DEFAULT NOW()); -- Trigger: otomatik profil oluştur CREATE OR REPLACE FUNCTION handle_new_user() RETURNS TRIGGER AS $$ BEGIN INSERT INTO profiles(id,email) VALUES(NEW.id,NEW.email); RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER; CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION handle_new_user(); -- RLS ALTER TABLE profiles ENABLE ROW LEVEL SECURITY; ALTER TABLE pharmacies ENABLE ROW LEVEL SECURITY; ALTER TABLE medicines ENABLE ROW LEVEL SECURITY; ALTER TABLE stock ENABLE ROW LEVEL SECURITY; ALTER TABLE activity_logs ENABLE ROW LEVEL SECURITY; ALTER TABLE search_analytics ENABLE ROW LEVEL SECURITY; CREATE POLICY "Herkes okur" ON pharmacies FOR SELECT USING (is_active=TRUE); CREATE POLICY "Herkes okur" ON medicines FOR SELECT USING (TRUE); CREATE POLICY "Herkes okur" ON stock FOR SELECT USING (TRUE); CREATE POLICY "Kendi profil" ON profiles FOR ALL USING (auth.uid()=id); CREATE POLICY "Auth log" ON activity_logs FOR INSERT WITH CHECK (auth.uid() IS NOT NULL); CREATE POLICY "Arama ekle" ON search_analytics FOR INSERT WITH CHECK (TRUE); CREATE POLICY "Admin log" ON activity_logs FOR SELECT USING ( (SELECT role FROM profiles WHERE id=auth.uid())='admin'); -- Service Role için (admin işlemleri API üzerinden yapılır)

3. Admin hesabı:

Supabase Dashboard → Authentication → Users → "Add User" ile admin e-postasını ekleyin.
Ardından SQL Editor'de: UPDATE profiles SET role='admin' WHERE email='admin@sizin.com';

🏥 Yeni Eczane Ekle

⚠️ Eczane için yeni bir Supabase kullanıcısı oluşturulacak. Bu işlem Service Role Key gerektirir.
Service Role Key'i CONFIG bölümüne eklemeyi unutmayın.
